# お天気予報アプリを作って学んだこと

OpenWeatherMap API を使った天気予報 Web アプリを、開発から本番公開まで一通り作りました。さらに「お気に入り都市の保存」「雨雲レーダー」「AIによる服装・持ち物提案」の3機能を追加実装しています。使った技術と工夫した点、学びをまとめます。

- 公開URL: https://weather-forecast-app-kappa-sand.vercel.app
- リポジトリ: https://github.com/toranana2162-web/weather-forecast-app

## 使用技術一覧

### 言語

| 言語 | 用途 |
|---|---|
| **TypeScript** | アプリ全体（型安全な開発） |
| **CSS** | スタイリング（グラスモーフィズム＋天気別グラデーション） |
| **JSX/TSX (HTML)** | React コンポーネントのマークアップ |

### フレームワーク・ライブラリ

| 技術 | 役割 |
|---|---|
| **Next.js 15（App Router）** | フロント＋サーバーAPIを統合したフルスタックフレームワーク |
| **React 19** | UI コンポーネント・状態管理（Hooks） |
| **Leaflet** | 雨雲レーダーの地図描画（OpenStreetMap ベース） |
| **@anthropic-ai/sdk** | AI服装提案で Claude API を呼び出す公式SDK |

### データベース

- **永続DBは不使用**。天気データは毎回 OpenWeatherMap API から取得。
- **お気に入り都市**だけはブラウザの **localStorage** に保存（サーバー不要・端末内で完結）。複数端末同期が必要になった段階で初めて DB（Supabase / PostgreSQL など）を検討する想定。

### その他ツール・サービス

| ツール | 用途 |
|---|---|

| **Claude API（`claude-opus-4-8`）** | AIによる服装・持ち物提案の生成 |
| **Vercel** | 本番ホスティング＋CI/CD＋暗号化環境変数 |

## 主な機能

| 機能 | 概要 |
|---|---|
| 都市検索 / 現在地 | 都市名検索と📍現在地ボタンで天気を表示 |
| カレンダー予報 | 6日分（当日＋5日）を日付で切り替え、3時間ごとの詳細も表示 |
| ⭐ お気に入り都市の保存 | localStorage に保存。チップから即座に切り替え、★ボタンで追加/削除 |
| 🌧️ 雨雲レーダー | Leaflet 地図に「現在（リアルタイム）」の降水を重ねて表示 |
| 🤖 AIによる服装・持ち物提案 | Claude が天気をふまえて服装・持ち物を提案 |

## 工夫したポイント

1. **API キーをブラウザに一切露出させない設計**
   OpenWeatherMap も Claude も、キーはすべて **Next.js のサーバー側 API ルート**（`app/api/...`）でのみ使用。`process.env` から読み、ブラウザには渡しません（BFFパターン）。

2. **雨雲レーダーのタイルもサーバー経由でプロキシ**
   地図タイルは `tile.openweathermap.org/.../{z}/{x}/{y}.png?appid=KEY` のようにキーが必要ですが、ブラウザから直接読むとキーが漏れます。そこで `app/api/tiles/[layer]/[z]/[x]/[y]` でサーバーが取得して中継し、キーを秘匿。受け付けるレイヤーは許可リスト、座標は数値のみと検証しています。

3. **AI提案は「構造化出力」で安定化**
   Claude API の `output_config.format`（JSON Schema）で、必ず `{summary, clothing[], items[]}` の形のJSONが返るよう制約。UIが崩れず安全にレンダリングできます。コストを抑えるため `effort: low`、ボタン押下時のみ呼び出し（自動でトークンを消費しない）。

4. **「いつの雨雲か」を明確化**
   雨雲レーダーは予報ではなく**実況（リアルタイム）**。カレンダーの選択日と混同しないよう、`● LIVE 実況` バッジとその都市の**現在時刻**（現地時間・1分ごと更新）を表示し、凡例にも明記。

5. **2つのAPIを並列取得して高速化** … 「現在の天気」と「予報」を `Promise.all` で同時取得。

6. **タイムゾーンのズレ対策** … UTC秒＋都市の `timezone` オフセットで現地時刻を正しく算出。

7. **3時間ごとの予報を「日別」に集計** … 最高/最低気温・平均湿度・最大降水確率を算出し、正午に近い時刻を代表アイコンに採用。

8. **天気に応じて背景が動的に変化** … 晴れ/曇り/雨/雷/雪/霧/夜でグラデーションを切り替え。

9. **型安全な設計** … API レスポンスの形を `lib/types.ts` で定義し、フロント・サーバー間で共有。

10. **Leaflet を React 19 で安全に使う** … `window` 前提の Leaflet を `useEffect` 内で動的 import し、SSR エラーを回避。地図はパネルを開いたときだけ生成・閉じたら破棄。

## 学んだこと

- **責務の分離とBFF**：秘密情報を扱う処理はサーバー（API ルート）に寄せる。地図タイルのような「画像」さえもキーを守るためにプロキシする、という発想。
- **LLM をプロダクトに組み込む勘所**：構造化出力でレスポンス形を保証する、課金を意識してボタン起動＆低effortにする、キー未設定でもアプリ全体は壊さずグレースフルに劣化させる。
- **シークレット管理とローテーション**：環境変数化はもちろん、漏れたキーの再発行・差し替え・旧キー削除まで実践。変数名の取り違えで本番が落ちる怖さも経験。
- **外部APIデータの正規化／タイムゾーンの罠**：生データをUIに合う形へ整形し、時刻は「どの基準か」を常に意識する。
- **CI/CD の便利さ**：`git push` で Vercel が自動ビルド＆公開。
